## 一、产品定位

一款微信小程序，帮助老年人/任何人通过语音讲述自己的故事，AI 自动梳理成时间线，整合照片和细节，最终生成图文并茂的个人传记，可打印成书作为礼物。

## 三、功能模块划分

### 模块 A：用户与故事管理
- 微信授权登录
- 创建故事（标题、主人公、简介）
- 故事列表（我的故事、家人的故事）
- 多主人公支持（一个故事可包含多人）

### 模块 B：语音访谈
- 微信原生录音（支持暂停/继续）
- 单次录音上限 5 分钟
- 录音列表管理
- AI 实时引导提问（可选）
- 录音转文字（异步处理）

### 模块 C：AI 时间线梳理 ⭐ 核心功能
- 从对话文本中自动提取事件
- 时间轴可视化（按年/月/日展示）
- 事件自动识别：时间、地点、人物、情感
- 手动拖拽调整事件顺序/时间
- 合并/拆分事件
- 标记不确定的时间（"大约 1980 年"）

### 模块 D：多媒体整合
- 从手机相册选择照片
- 照片关联到具体事件
- 照片裁剪/滤镜（轻量）
- 人物标记（照片中是谁）

### 模块 E：AI 润色与写作
- 原始对话 vs 润色后对比
- 多种风格可选（朴实、文艺、口语化）
- 文学性增强（描写、比喻、情感渲染）
- 段落重写/续写
- 反转/戏剧化处理（可选）

### 模块 F：预览与导出
- 时间线滚动预览
- 图文混排的传记预览
- 导出为 PDF（可打印格式）
- 对接印刷服务（预留接口）

## 四、数据模型设计

### 4.1 用户 (User)
```
{
  id: string          // 微信 openid
  nickName: string    // 微信昵称
  avatarUrl: string   // 头像
  created_at: datetime
}
```

### 4.2 故事 (Story)
```
{
  id: string
  userId: string          // 创建者
  title: string           // 故事标题，如"父亲的回忆录"
  protagonist: string     // 主人公姓名
  coverImage: string      // 封面图 URL
  description: string     // 简介
  status: enum            // draft, recording, processing, completed
  created_at: datetime
  updated_at: datetime
}
```

### 4.3 录音片段 (AudioClip)
```
{
  id: string
  storyId: string
  duration: number        // 时长（秒）
  audioUrl: string        // 录音文件 URL
  transcript: string      // 转文字结果
  transcriptStatus: enum  // pending, processing, done, failed
  created_at: datetime
}
```

### 4.4 事件 (Event) ⭐ 核心模型
```
{
  id: string
  storyId: string
  title: string           // 事件标题，如"第一次进城"
  content: string         // 事件详细描述（润色后）
  rawContent: string      // 原始对话原文
  date: string            // 日期，如 "1985-03" 或 "约1980年"
  datePrecision: enum     // exact, year_only, month_only, approximate
  sortOrder: number       // 手动排序
  location: string        // 地点
  people: string[]        // 相关人物
  tags: string[]          // 标签
  emotion: string         // 情感标签
  photos: PhotoRef[]
  created_at: datetime
}
```

### 4.5 照片 (Photo)
```
{
  id: string
  storyId: string
  eventId: string         // 关联事件（可为空）
  url: string
  caption: string         // 照片说明
  peopleTagged: string[]  // 照片中的人物
  created_at: datetime
}
```

### 4.6 传记版本 (Biography)
```
{
  id: string
  storyId: string
  title: string
  content: string         // 完整传记文本（含图片引用）
  style: string           // 风格
  pdfUrl: string          // 导出 PDF 地址
  status: enum            // draft, generating, completed
  created_at: datetime
}
```

## 八、API 接口设计（核心）

```
POST   /api/story                   创建故事
GET    /api/story/list              获取故事列表
GET    /api/story/:id               获取故事详情
PUT    /api/story/:id               更新故事
DELETE /api/story/:id               删除故事

POST   /api/audio/upload            上传录音
POST   /api/audio/transcribe        录音转文字
GET    /api/audio/list/:storyId     获取录音列表

POST   /api/event/extract           从文本提取事件（AI）
GET    /api/event/list/:storyId     获取时间线事件列表
PUT    /api/event/:id               更新事件
POST   /api/event/reorder           调整事件顺序
POST   /api/event/merge             合并事件
POST   /api/event/split             拆分事件

POST   /api/photo/upload            上传照片
PUT    /api/photo/:id               更新照片信息
DELETE /api/photo/:id               删除照片

POST   /api/biography/generate      生成传记（AI）
GET    /api/biography/:id           获取传记内容
POST   /api/biography/export-pdf    导出 PDF

POST   /api/ai/polish               润色文字
POST   /api/ai/interview-ask        生成追问问题
POST   /api/ai/extract-people       提取人物信息
```

## 九、测试策略

### 分层测试
1. **单元测试** — 每个函数/接口单独测试
2. **集成测试** — 多模块协作测试
3. **AI 输出测试** — 验证 AI 输出的格式和质量
4. **UI 测试** — 小程序页面交互测试

### 核心测试用例示例
```
时间线提取测试：
  - 输入：一段关于"1985年去北京打工"的杂乱回忆
  - 验证：AI 能否正确提取出时间、地点、事件
  - 通过标准：时间准确、事件描述合理

润色测试：
  - 输入：一段口语化回忆
  - 验证：润色后保留原意，文学性提升
  - 通过标准：不改变事实，语言更优美

录音转文字测试：
  - 输入：一段普通话录音
  - 验证：转文字准确率 > 90%
```
