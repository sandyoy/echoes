## 话题三：长期记忆与存档机制确立

**背景：** 用户指出 AI 在两个问题上存在不足：
1. 多议题跳跃讨论时容易丢失上下文
2. 用户需要在微信上能快速找到与 Hermes 的对话窗口

**小鲸鱼的方案（用户转述）：**
- 三个记忆仓库：memory（2200字符）、session_search（无限但只能看摘要）、docs/（无限）
- 建议用户说"收工，存档"时执行存档
- 按日期写日记保存完整讨论（不分拆话题），存到 docs/ 下
- memory 里存索引

**Hermes 的优化方案（被采纳）：**
- **取消口号机制**：每次聊到最后 Hermes 主动问"还有别的事吗？"，用户说"没了"就自动存档
- **增加 INDEX.md**：在 `docs/daily/` 下维护索引表格，避免 memory 溢出
- memory 只存一句话：`项目讨论记录索引见 docs/daily/INDEX.md`
- 日记格式结构化：按话题分段，记录背景、讨论过程、决策、待办
- 写一个 skill 让存档流程自动执行

**小鲸鱼的约定：** 小鲸鱼也会采用同样的方案。

**用户最终决策：** 同意该方案，即日起执行。
