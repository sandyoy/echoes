## 话题二：速记工具 + 医生AI助理 + 圆爱康知识库（三位一体）

### 讨论过程
从"聊天时记笔记"的需求出发，逐步延展为三个产品层次：

**通用速记工具（底座）：**
- 录音→转文字→提取要点
- 关键创新：**前情介绍机制**——录音前说清楚背景和关注点，提取才准
- 过渡方案：sandy 手机录好音发过来，小鲸鱼帮忙处理
- 最终版：微信小程序，点"开始记"就自动录音

**产品线A：AI医生助理（给医生个人免费使用）：**
- 医生打开工具 → 设定"我是妇产科医生" → 跟患者问诊录音 → 自动生成诊疗记录+治疗方案
- 可以拍照贴图（病历、检查报告）
- 线索存在云端，每个医生一个账号
- **免费给医生用**，核心目标：让医生觉得"懂他、好用"
- 变现不在工具本身，而在医生资源→圆爱康的业务协同
- 医生不只是医生，是"用户"——任何有想法的人都能用（做课题、做方案）

**产品线B：圆爱康智能康复系统：**
- B1：医患居家沟通录音 → 自动提取患者指标 → 生成康复计划草稿 → 医生确认推送到患者端
- B2：六师共管服务群对话 → 自动提取QA → 沉淀为知识库
- 最终形成"数据飞轮"——每次沟通都让系统更聪明
- 包含用药管理——医生口头说的用药建议自动转为患者可见的用药提醒

### 用户偏好/顾虑捕捉
- **变现路径：** 往事可追忆的变现不在线，在线免费，变现在线下打印实体纪念册 → 数据在云端，变现在线下
- **知识库必须先建好**，否则工具输出的质量不够
- **不能用通用的AI提取**（豆包不准的原因就是没有上下文+不懂用户）

### 优先级排序（sandy确认）
1. 通用速记工具（底座）
2. 产品线A：医生AI助理
3. 产品线B：圆爱康智能康复系统
