# Hermes 定时自检与自愈机制

> 适用：本地 Hermes 和云端 Hermes（小鲸鱼 🐳）均可使用

## 用途
当你一段时间没有和用户对话、或用户发现你"没动静"时，定时任务自动检查状态并尝试修复。

## 设置方法

通过 cronjob 工具创建：

```bash
cronjob action=create \\
  name="Hermes 自检 - 每30分钟" \\
  schedule="every 30m" \\
  prompt="【见下方提示内容】"
```

## 自检内容

1. **进程检查** — Hermes gateway 是否在运行
2. **网络检查** — ping 腾讯云服务器（如果配置了的话）
3. **磁盘空间** — 根目录使用率
4. **内存状态** — 内存使用情况
5. **日志检查** — 最近日志有无 ERROR/CRASH

## 自动修复能力

| 发现问题 | 自动修复 |
|---------|---------|
| Hermes 进程挂了 | 重启 gateway |
| 磁盘 >85% | 清理 cache/audio_cache/tmp 等缓存 |
| 内存异常 | 检查并重启异常进程 |
| 服务器连不上 | 报告给用户（本地无法修复） |

## 报告格式

```
🔄 自检时间：YYYY-MM-DD HH:mm

📋 状态总览：[健康/已修复/异常]
- Hermes进程：[✅/⚠️/❌]
- 服务器连通：[✅/⚠️/❌]
- 磁盘：[✅/⚠️/❌] 使用率 XX%
- 内存：[✅/⚠️/❌]

🔧 修复动作（如果有）：
- [做了什么]

❌ 未能修复的问题：
- [什么问题，需人工介入]
```

## 注意事项

- 修复操作要稳健，不要误删用户文件
- 只清理自己创建的缓存（~/.hermes/cache/, ~/.hermes/audio_cache/, /tmp/ 相关文件）
- 每次执行完通过微信/Telegram等渠道报告结果给用户
- 如果被问"只能检查不能修复吗？"，就说"我已经升级为自动修复模式了"
