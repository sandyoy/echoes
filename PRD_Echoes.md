## 2. 一句话定义

> **Echoes = 你的人生时间轴。** 像手机相册按时间排列照片一样，它把你的语音、文字、照片、康复数据…按时间排列成"人生"。

## 3. 三层产品架构

```
               ┌─────────────────────────────────────┐
               │    Layer 1：人人可用的人生记录工具    │
               │  语音回忆 · AI采访 · 时间轴 · 画册   │
               │  目标用户：会用手机的老人、子女、     │
               │  任何想记录人生的人                   │
               └─────────────────────────────────────┘
                              ↓
               ┌─────────────────────────────────────┐
               │    Layer 2：居家康复患者的数据档案    │
               │  康复日记 + 物联网设备数据 + 检查报告  │
               │  圆爱康居家康复服务附赠                 │
               └─────────────────────────────────────┘
                              ↓
               ┌─────────────────────────────────────┐
               │    Layer 3：硬件延伸                  │
               │  一键按钮设备 → 老人不用手机也能记录   │
               │  AI眼镜版本 → 未来形态                │
               └─────────────────────────────────────┘
```

**关键设计原则：Layer 1 是底座，Layer 2 和 Layer 3 是叠加。**

- Layer 1 不依赖 Layer 2/3，独立可用
- Layer 2 的康复数据自动汇入 Layer 1 的时间轴——患者在康复，也在记录人生
- Layer 3 是为"完全不会用手机的人"准备的入口

## 4. 用户画像（完整版）

### 4.1 核心用户：能使用智能手机的老人
- **画像：** 60-80岁，会用微信发语音、刷短视频、看朋友圈
- **痛点：** 记忆碎片化在朋友圈里，没有工具能把它们变成完整的人生故事
- **场景：** 社区食堂遇到、养老院、居家独居
- **他们是怎么被触达的：** 子女推荐 / 社区活动推广 / 居家康复服务附赠

### 4.2 用户代理：子女
- **画像：** 30-50岁，在外工作，想让父母留下一本人生故事
- **行为：** 帮父母安装，教父母用，最后下单打印画册送给父母
- **痛点：** 父母的事迹记在心里，但没有工具帮父母系统地记录下来

### 4.3 使用者+生产者：故事创作者
- 需要真实故事素材的编剧、短视频团队、AI训练公司
- 这个群体**不是Echoes的直接使用者**，而是长远的数据买方

### 4.4 未来用户：不会用智能手机的老人
- 一键按钮硬件 → 按下去开始录音 → 松手停止 → 自动上传
- AI眼镜 → 未来形态

## 7. 边界与限制

- **隐私合规：** Echoes 同时记录"人生回忆"（Layer 1）和"医疗康复数据"（Layer 2），这两类数据的隐私要求不同。康复数据受《个人信息保护法》和《健康医疗大数据管理规定》约束，不能和普通回忆数据混用。**建议：Layer 1 用户自己控制数据可见性，Layer 2 的康复数据单独加密存储。**
- **技术限制：** 微信小程序录音最长60秒/次（可以分段），Web无此限制
- **备案限制：** 微信小程序正式上线需要ICP备案通过 + HTTPS
- **老人使用限制：** UI必须极大、极简、无需打字即可操作

## 8. 交互流程

### 场景A：老人自述回忆
```
打开Echoes → 首页看到大大的"按住说话"按钮 →
按住说一段回忆（比如"我小时候家门口有棵槐花树"）→ 
松开 → 系统转成文字 → 弹出编辑框确认 →
用户改几个字或直接点"保存" → 回忆出现在时间轴 →
下次打开时间轴就能翻到这条
```

### 场景B：AI采访（子女引导老人）
```
子女帮老人打开 → 切换到"AI采访" → 
AI："您好呀，我是您的回忆小助手。今天想聊聊哪段时光呢？"
老人用语音回答 → AI顺着话题继续问 →
像两个人在聊天一样 → 采访完了自动生成一条时间轴记录
```

### 场景C：打印画册
```
在时间轴上选"生成画册" → 
系统自动把所有回忆按时间顺序排版 →
预览效果 → 确认下单 → 支付¥69 → 
等3-5天收到实体书
```

### 场景D（未来）：居家康复患者的Echoes
```
患者做完康复项目 → 圆爱康赠送Echoes使用权限 →
物联网康复设备的数据自动同步到时间轴 →
患者自己也可以语音记录每天的康复感受 →
时间轴变成了"康复日记+人生回忆"的混合 →
医生能查看患者这段时间的整体康复历程
```
